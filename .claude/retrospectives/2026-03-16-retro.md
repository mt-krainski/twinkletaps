## What could be improved

### Theme A: Command chaining/pipe violations (7/11 sessions)
`cd /path && agent-utils git-commit`, `cmd 2>&1 | tail -40`, `npm run lint 2>&1 | grep`. In GFD-83, this caused a full session termination. The CLAUDE.md clarification ("check for chaining operators first") was added mid-week but violations continued.

### Theme B: `sleep` for CI polling (5/11 sessions)
`sleep 60/120/180` followed by `gh-pr-checks` poll loops. GFD-52 had 10+ sleep-poll cycles. One session (GFD-53) self-corrected to `run_in_background` in round 2.

### Theme C: TDD not strictly followed (6/11 sessions)
Tests written but no explicit "run it red" step. Only GFD-84 and GFD-72 had clear red-green-refactor cycles.

### Theme D: `agent-utils gh-pr-reply` syntax (3/11 sessions)
Wrong argument order guessed each time (GFD-67: 4 attempts, GFD-72: 2 attempts, GFD-92: chaining confusion).

### Theme E: Code reviewer missing CLAUDE.md violations (2/11 sessions)
GFD-51: `process.env` used directly. GFD-92: `describeIfReady` (conditional test skip) introduced. Both caught only by human review.

### Theme F: Known memory rules violated (2/11 sessions)
GFD-64: assignee `{"name": ...}` despite memory rule. GFD-92: `describeIfReady` despite `feedback_never_skip_tests.md`.
