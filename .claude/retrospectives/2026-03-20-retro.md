## Agreed improvements

### 1. PreToolUse chaining lint hook (`.claude/hooks/reject-chaining.sh`)
Added a `PreToolUse` hook on `Bash` that rejects commands containing `&&`, `||`, `|`, `$()`, or backticks with a clear message: "Split into separate Bash calls instead." Runs before the bash-allowlist hook. This catches the #1 violation at the source with an unambiguous error message, instead of relying on the permission system's confusing denial.

### 2. Softer commit guidance in `/execute` Step 3
Changed "Commit frequently" to explicit guidance: commit after each logical unit (passing test + implementation, new component, config change). Notes that WIP commits are fine — they'll be squashed at merge time. Protects against work loss from session crashes.

### 3. Re-run full `/wrap` on resume
Added instruction at the top of `/wrap`: if resuming after interruption, restart from Step 1. Simpler and safer than tracking which steps completed.
